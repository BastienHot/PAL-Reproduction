# External Downloads Required

This document lists everything that must be downloaded from sources other than the
PAL and ESConv GitHub repositories before the code can run.

The old `transformers==4.9.2` version means the HuggingFace Hub API may not work
for automatic downloads. Most model weights should be downloaded manually.

---

## 1. Model Weights

### BlenderBot-small-90M (required for PAL model)
- **What**: Pre-trained BlenderBot-small model weights (`pytorch_model.bin`)
- **Where to place**: `codes/Blenderbot_small-90M/pytorch_model.bin`
- **Source**: https://huggingface.co/facebook/blenderbot_small-90M
- **Note**: The tokenizer files (config.json, merges.txt, vocab.json) are already
  included in the repo. Only the model weights file is missing.
- **How to download**:
  ```bash
  # Option 1: Direct download
  wget https://huggingface.co/facebook/blenderbot_small-90M/resolve/main/pytorch_model.bin \
       -O codes/Blenderbot_small-90M/pytorch_model.bin

  # Option 2: Using huggingface_hub (if installed)
  python -c "from huggingface_hub import hf_hub_download; hf_hub_download('facebook/blenderbot_small-90M', 'pytorch_model.bin', local_dir='codes/Blenderbot_small-90M')"
  ```

### BART-large-CNN (required for persona extractor)
- **What**: Pre-trained BART model fine-tuned on CNN/DailyMail summarization
- **Where to place**: Auto-cached by HuggingFace transformers, or download manually
- **Source**: https://huggingface.co/facebook/bart-large-cnn
- **Used by**: `persona_extractor/train_bart.py` and `codes/RUN/interact_strat.sh`
- **Note**: This is a ~1.6GB model. With `transformers==4.9.2`, auto-download
  *should* still work via `from_pretrained("facebook/bart-large-cnn")`.
  If it fails, download manually:
  ```bash
  # Download all files to a local directory
  mkdir -p persona_extractor/bart-large-cnn
  cd persona_extractor/bart-large-cnn
  wget https://huggingface.co/facebook/bart-large-cnn/resolve/main/pytorch_model.bin -O pytorch_model.bin
  wget https://huggingface.co/facebook/bart-large-cnn/resolve/main/config.json -O config.json
  wget https://huggingface.co/facebook/bart-large-cnn/resolve/main/tokenizer.json -O tokenizer.json
  wget https://huggingface.co/facebook/bart-large-cnn/resolve/main/vocab.json -O vocab.json
  wget https://huggingface.co/facebook/bart-large-cnn/resolve/main/merges.txt -O merges.txt
  ```
  Then update `hparams.model_dir_or_name` in `train_bart.py` to point to this directory.

### BART-base tokenizer (required for persona data processing)
- **What**: BART-base tokenizer (used only for special token definitions during
  PersonaChat preprocessing)
- **Source**: https://huggingface.co/facebook/bart-base
- **Used by**: `persona_extractor/process_bart_df.py`
- **Note**: Only the tokenizer is used, not the model weights. Auto-download
  should work with `BartTokenizer.from_pretrained('facebook/bart-base')`.

### SimCSE (required for cosine similarity evaluation only)
- **What**: Sentence embedding model for computing response-persona similarity
- **Source**: https://huggingface.co/princeton-nlp/sup-simcse-bert-base-uncased
- **Used by**: `codes/get_cos_similarity.py`
- **Note**: Only needed for the cos-sim evaluation metric, not for training or
  inference. Auto-download does **not** work with `transformers==4.9.2`.
  Download the model manually (requires `git-lfs`):
  ```bash
  sudo apt install git-lfs   # if not already installed
  cd codes
  git clone https://huggingface.co/princeton-nlp/sup-simcse-bert-base-uncased simcse-bert-base-uncased
  cd simcse-bert-base-uncased && git lfs pull && cd ..
  ```
  Then pass `--simcse_model ./simcse-bert-base-uncased` to `get_cos_similarity.py`.

## 2. Datasets

### PersonaChat (required for persona extractor training)
- **What**: PersonaChat dataset with both persona annotations
  (`{train,valid,test}_both_original.txt`)
- **Where to place**: `persona_extractor/data/`
- **Source Option A** (author's Google Drive, recommended — includes test split):
  https://drive.google.com/drive/folders/1-lnO2xi9YLr82VdcF5oVlQClHsRnn9EP?usp=sharing
  (See PAL GitHub issue #3)
- **Source Option B** (ParlAI):
  ```bash
  # Install ParlAI and download
  pip install parlai
  python -c "import parlai.core.build_data as build_data; build_data.download('personachat')"
  ```
  Note: The ConvAI2/ParlAI version may not have a test split. The Google Drive
  version from the authors is more reliable for reproduction.
- **Expected files after download**:
  ```
  persona_extractor/data/
  ├── train_both_original.txt
  ├── valid_both_original.txt
  └── test_both_original.txt
  ```

### PESConv.json (already included)
- The persona-augmented ESConv dataset is already included at
  `codes/_reformat/PESConv.json` (copied from the PAL repo).
- This is ESConv + persona annotations generated by the authors' trained
  persona extractor.

### ESConv.json (optional, for baseline experiments)
- **What**: Original ESConv dataset (no persona annotations)
- **Source**: Already in your project at `Emotional-Support-Conversation/ESConv.json`
- **If needed**: Copy to `codes/_reformat/ESConv.json`
- **Used when**: Running `_reformat/process.py --add_persona False`

## 3. Evaluation Resources

### GloVe embeddings (required for word2vec-based evaluation metrics)
- **What**: GloVe 6B 300d word embeddings
- **Where to place**: `codes/metric/word2vec/`
- **Source**: https://nlp.stanford.edu/projects/glove/
  (Download `glove.6B.zip`, extract `glove.6B.300d.txt`)
- **Conversion** (converts GloVe text → word2vec text → gensim binary):
  ```bash
  # 1. Download and extract the GloVe vectors
  wget https://nlp.stanford.edu/data/glove.6B.zip -O glove.6B.zip
  unzip glove.6B.zip glove.6B.300d.txt

  # 2. Copy to the word2vec directory and run the conversion script
  cp glove.6B.300d.txt codes/metric/word2vec/
  cd codes/metric/word2vec
  python generate_w2v_files.py
  cd ../../..

  # 3. Clean up source files (the .bin is all that's needed at runtime)
  rm glove.6B.zip glove.6B.300d.txt
  ```
  This produces `glove.6B.300d.model.bin` (and associated files) which are
  loaded by `evaluate.py` at inference time.

### NLTK data (required for tokenization)
- **What**: NLTK punkt tokenizer
- **How**:
  ```python
  import nltk
  nltk.download('punkt')
  ```

### Java 11 (required for METEOR metric)
- **What**: Java runtime for the METEOR JAR
- **Note**: Java 11 is recommended. Newer versions may not work with `meteor-1.5.jar`.
  ```bash
  sudo apt install openjdk-11-jre -y
  ```

### METEOR paraphrase data (required for METEOR metric)
- **What**: English paraphrase table and data files for METEOR
- **Where to place**: `codes/metric/pycocoevalcap/meteor/data/`
- **Note**: These files are **not** auto-downloaded and must be extracted from the
  METEOR 1.5 release:
  ```bash
  cd /tmp
  wget https://github.com/cmu-mtlab/meteor/releases/download/v1.5/meteor-1.5.tar.gz
  tar xzf meteor-1.5.tar.gz
  cp -r meteor-1.5/data codes/metric/pycocoevalcap/meteor/
  rm -rf meteor-1.5 meteor-1.5.tar.gz
  ```

## 4. Pre-trained Persona Extractor Checkpoint (optional)

If you want to skip training the persona extractor from scratch:
- The authors reference checkpoint `epoch=7-step=8384.ckpt`
- This checkpoint is **NOT** available in the repository or online
- You must train it yourself using `persona_extractor/train_bart.py`
- Training takes ~10 epochs on PersonaChat with BART-large-CNN
